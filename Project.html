Topic

The team proposed to design a humidity and temperature monitor using the Arduino kit. The project will utilise the Arduino nano microprocessor and the small electrical components such as LCD unit for displaying the temperature and humidity to its user, temperature sensor, humidity sensor, 9V battery pack available off-the-shelf at JAYCAR electrical goods store.

All the components will be assembled using the prototyping breadboard and small connector cables to all for easy assembly. The code to run the components will be coded on Arduino IDE and then uploaded on to the microprocessor, the expected battery life is up to 2 weeks.

The chatbot component of the project is designed to streamline the process of monitoring the Arduino and data generated.The scope of this program will enable the user to type questions and get automated responses with information retrieved from the Arduino board. Some of the data that would be retrieved for the user includes the following:

Current temperature

Temperature estimations for the next week (using weather API’s)

Getting automated message alerts when temperatures exceed a certain amount

Current percentage of humidity

Retrieve the estimated humidity levels for the next week (with appropriate API’s)

Applications

This project whilst being simple has many practical applications most notably in agriculture where it could be deployed to monitor humidity levels and other temperature information which would assist farmers in keeping track of their crops.

Other cases where it could be useful is for specialised machinery or instruments that must be kept at a constant temperature. This chatbot would be able to quickly alert the user if any temperatures changed. 

Motivation

The motivation behind this simple project is education. Although the project is simple enough for a team to build within a reasonable amount of time, the goal of this project is to educate high school children and get them excited about information technology and software/hardware engineering.

The temperature and humidity monitor and the chatbot are interactive and easy for high school children to build over a few sessions which will allow them to gain an appreciation for technology and help guide the interested children to the science and technology pathway.

With this project we hope to expose the high school children to minor coding, configuring small electrical components and make it a team building activity.

Landscape

There are many similar and advanced programs and gadgets out there today, from the likes of amazon’s echo dot to google’s home assistant and even most phones, available to everyone these days, are capable of doing the things outlined within this project.

But that is missing the point, if something is available in store doesn’t mean you should go the easy route and just buy it off the shelfs. If everything did that then there would be no creation and innovation, imagine the first people who built smartphones kept buying the old non-smartphones and didn’t think it was worth building something of their own and make a few changes.

Similarly, if we can start somewhere small, get more people excited about building their own little gadget/toys to be excited and proud of, who knows, skys the limit for creativity and innovation.

Scope and limits

Arduino unit - Arduino Designer

Scope

We aimed to build an arduino based weather and humidity sensor along with a LCD display to allow the user to get relevant weather information of their surroundings and also in conjunction with the the chatbot get the live weather data for your suburb or locality by asking questions to the chatbot.

The arduino temperature and humidity sensor uses basic informational technology and some aspects of electronics to assist the members of the public affected by bushfires and other disasters occuring in the world.

Also helps young minds get acquainted with simple technology  showing them a potential career path and encouraging their inner creativity to shine.

Limitations

Current limitations on the arduino is that it has a pretty basic screen which means the user interface is quite unfriendly. Next, the unit runs off battery power which means constant change of battery unit is required for continuous use.

Furthermore, there isn’t a link to the chatBot at this time as these two seperate items are being prototyped at different locations.

Plans for future expandability

As the current project is pretty basic, the next step if we were given more funding and resources we’d like to look to hire qualified electrical engineers to perfect the design, make it more energy efficient and this will allow us to use better components for the prototype.

Furthermore, a better design in turn means that the demand for the product will increase and be capable of helping and educating a wider range of people.

Accompanying chatbot - ChatBot Designer

Scope

The chatbot in its current form is a command-line Python application which contains some demonstration questions and answers hardcoded into the application.

The application then will talk back to the user based on the input provided with the gradual goal of integrating with the Arduino board to retrieve data from the temperature and humidity sensor and extend functionality through third party API’s. 

Current limitations

The major limitation at this stage with the chatbot is the fact it requires datasets to be imported for it to learn and improve its accuracy. Another limitation is the lack of integration at the current time between our chatbot and the Arduino system assembled by Vignesh. 

Plans for future expandability

In its current format the chatbot is very basic as a means to merely demonstrate our idea however with some future modifications the below features could be implemented making our application far more feature rich. 

SMS integration (Twilio api)

Ability to recognize foreign languages (built in to module but requires additional datasets)

Improved conversational responses and questions from the chatbot

Integration of more datasets for better accuracy

Ability to convert celsius to fahrenheit (using third party API’s)

The ability to run as a headless application on a dedicated device to be accessed remotely

Analytics on all temperatures through a web interface containing graphs
